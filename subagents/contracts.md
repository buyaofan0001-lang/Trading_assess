# 四代理数据契约（复用版）

## 1. EvidenceItem（统一来源对象）
```json
{
  "id": "S1",
  "title": "来源标题",
  "url": "https://...",
  "level": "一级|二级|三级",
  "publisher": "发布主体",
  "publish_date": "YYYY-MM-DD",
  "access_checked_at": "YYYY-MM-DD",
  "subject_verified": true,
  "notes": "可选，主体一致性或口径说明"
}
```

硬约束：
- `url` 可访问，非 404/错误重定向页。
- `publish_date` 与 `access_checked_at` 必填且格式为 `YYYY-MM-DD`。
- 关键结论至少满足 `一级 + 二级` 双源交叉；仅三级证据时必须标注 `未证实/待验证`。

## 2. FactRow（风险/利好/利空通用行）
```json
{
  "编号": "R1|P1|B1",
  "事实": "...",
  "影响": "...",
  "日期": "YYYY-MM-DD",
  "来源编号": "S1",
  "类别": "财务|行业景气|政策|公司公告"
}
```

## 3. 三层结论对象（事件/技术/因果）
```json
{
  "事件层": {"状态": "已证实|部分证实|未证实/待验证", "结论": "...", "来源": ["S1"]},
  "技术层": {"状态": "已证实|部分证实|未证实/待验证", "结论": "...", "来源": ["S2"]},
  "因果层": {"状态": "已证实|部分证实|未证实/待验证", "结论": "...", "来源": ["S3"]}
}
```

## 4. 宏观行业研判官输出
```json
{
  "宏观清单": {
    "风险": ["FactRow x >=8"],
    "利好": ["FactRow x >=8"]
  },
  "行业深研": {
    "政策风向": {"结论": "...", "证据来源": ["S1", "S2"]},
    "行业景气度": {"结论": "...", "证据来源": ["S3"]},
    "未来增值空间": {"结论": "...", "证据来源": ["S4"]}
  },
  "基本面分析检查": {
    "收入结构": {"结论": "...", "证据来源": ["S5"], "日期": "YYYY-MM-DD", "统计口径": "TTM|单季|累计"},
    "利润质量": {"结论": "...", "证据来源": ["S6"], "日期": "YYYY-MM-DD", "统计口径": "TTM|单季|累计"},
    "现金流质量": {"结论": "...", "证据来源": ["S7"], "日期": "YYYY-MM-DD", "统计口径": "TTM|单季|累计"},
    "估值与预期": {"结论": "...", "证据来源": ["S8"], "日期": "YYYY-MM-DD", "统计口径": "TTM|单季|累计"},
    "催化财务映射": {"结论": "...", "证据来源": ["S9"], "日期": "YYYY-MM-DD", "统计口径": "TTM|单季|累计"}
  },
  "三层结论矩阵": "三层结论对象",
  "sources": {"S1": "EvidenceItem", "S2": "EvidenceItem"}
}
```

## 5. 基本面检查门槛
- 维度固定且必须齐全：`收入结构`、`利润质量`、`现金流质量`、`估值与预期`、`催化财务映射`。
- 每个维度必须带：`结论 + 证据来源 + 日期 + 统计口径`。
- 任一关键维度 `未证实/待验证` 且直接关联交易主逻辑时，逻辑验证不成立。

## 6. 空头对抗审计官输出
```json
{
  "空头对抗": {
    "利空": ["FactRow x >=8"]
  },
  "coverage_gaps": [
    {"类别": "财务|行业景气|政策|公司公告", "缺口说明": "...", "补证需求": "..."}
  ],
  "三层结论矩阵": "三层结论对象",
  "sources": {"S1": "EvidenceItem"}
}
```

硬约束：
- 利空必须覆盖四类：`财务/行业景气/政策/公司公告`。
- 覆盖不足必须写入 `coverage_gaps`，不得静默。
