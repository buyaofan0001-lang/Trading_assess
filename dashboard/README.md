# 守门台 V1.5

个人交易控制看板初版。它先回答“今天允许做什么”，再展示持仓相对同行强弱、隔夜美股环境、盘后资金流与每日交易日记。

## 启动

在项目根目录运行：

```bash
python3 dashboard/server.py
```

浏览器打开 <http://127.0.0.1:8765>。

## 数据口径

- A股价格：项目统一入口 `tushare_client.py`，使用单代码 `rt_k` 轮询，缓存 25 秒。
- 当日同行折线：`/api/intraday` 提供 1 分钟不复权价格，前端统一换算为相对昨收涨跌幅；持仓线加粗高亮，午休区间单独压缩。主源为 AkShare/东方财富，Yahoo Finance 只作批量快速回退，较慢的东方财富完整分钟线在后台串行补齐，不阻塞页面。
- 海外同行：已映射的美股同行会同时显示在对应持仓表格底部，例如长电科技下的 AMKR、ASX。它们使用最近完整纽约交易日的收盘价和单日收益，并通过“隔夜海外同行”分隔；只作跨市场并排阅读，不进入A股同行中位数、排名或强弱结论。
- 5日/20日收益：Tushare 日线，缓存 30 分钟。
- 板块资金流：同花顺口径日频数据，只显示最近可用盘后日期。
- 美股：Yahoo Finance 最近完整交易日，缓存 30 分钟。
- 长电科技按用户本轮确认和工作簿 rows 194–198 记录为900股，加权成交成本99.854元；看板据此显示动态浮动盈亏。
- 同行与海外映射均是 V1 候选名单，需用户确认后锁定。

## 每日交易日记

- 左侧历史列表读取仓库根目录的 `日记/*.md`，按日期倒序展示；旧文件名即使月份或日期没有补零，也会归一为 `YYYY-MM-DD` 日期。
- 点击历史记录可直接读取原文，编辑后仍写回原文件；新日记统一保存为 `YYYY-MM-DD.md`，不会因旧命名不统一而生成重复记录。
- 日期输入、今天按钮、复盘提纲、未保存提示与 `Cmd/Ctrl + S` 快捷保存均在本地完成；内容不会上传到第三方服务。
- `/api/journals` 返回目录索引，`/api/journal?date=YYYY-MM-DD` 读取单篇，`POST /api/journal` 保存单篇。后端仅接受合法日期，限制500KB，并使用原子替换避免写到一半留下损坏文件。

## 刷新与性能策略

- A股交易时段页面可见时每30秒自动刷新；页面切到后台后暂停，重新可见时立即刷新。
- 折线图跟随自动刷新请求更新；东方财富分钟线约每60秒后台补齐一次。若各线最新时间不一致，图头显示时间范围；单只缺失不会清空整组。
- 15:05–16:30按5分钟轮询盘后数据；其他非交易时段暂停自动请求，仍可手动刷新。
- 5日/20日同行数据在首次加载后后台预热，切换时直接读取浏览器内存，不再重新组装美股和资金流。
- 后端并发获取不同证券，并对相同历史请求做单飞去重；手动刷新会清除A股实时快照缓存。

## 可配置项

编辑 `dashboard/config.json` 可调整持仓、同行候选与美股映射。Token 仍由根目录 `tushare_client.py` 统一初始化，不写入看板源码。

## 浏览器验收

首次运行：

```bash
cd dashboard
npm install
npx playwright install chromium
npm test
```

测试覆盖真实数据加载、两组9条同行分钟线与持仓高亮、海外同行篮子与排名隔离、红灯权限、5日切换、刷新、历史日记读取、编辑保存反馈、控制台错误和390px窄屏溢出。
