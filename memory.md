## 重点跟踪标的与历史核心逻辑

### 思源电气
- 属于高优先级长期跟踪标的。
- 历史核心逻辑是海外电网、电力设备出口与景气延续。
- 对国内电力需求逻辑相对保留，更看重海外需求弹性。
- 曾反复讨论中期目标空间与趋势延续性，持有周期预期偏长。
- 关注财报兑现、现金流解释、趋势是否被破坏，以及大涨后的止盈执行问题。

### 长电科技
- 属于高频交易与复盘标的。
- 核心逻辑是 AI 芯片、国产算力与先进封装需求映射到本土 OSAT。
- 反复出现的疑问是：行业方向可能没错，但利润释放是否足够快、是否足够强。
- 更关注“订单与利润映射是否被真实验证”，而不是只听概念刺激。
- 交易上明显结合趋势位、30 日线、放量突破等执行条件。

### 兆易创新
- 主要作为存储链条映射标的研究。
- 核心逻辑是存储景气是否继续、供需是否仍偏紧。
- 核心担忧是：
  - 受益层级是否足够高
  - 是否直接受益于高端 AI 存储需求
  - 业绩释放强度是否匹配市场预期
- 更偏向等待确认后再出手，不愿在逻辑未清前追涨。

### 亨通光电
- 核心关注点是 AI 带来的光通信、海缆、出口与订单。
- 重要历史教训：不能基于错误的低估值前提建立交易计划，估值口径必须先核实。

### 三花智控
- 关注机器人、液冷、热管理的多重映射。
- 需要同时满足产业逻辑与价格确认，不能只靠题材想象。

## 历史上反复出现的交易问题
- 容易先相信主题叙事，再去补证据。
- 容易把“产业方向对”误当成“这个时点买这只股票也对”。
- 容易在盈利阶段因为害怕卖飞而延迟止盈，导致利润回吐。
- 2026-04 光迅科技/华工科技复盘暴露出更具体的问题：浮盈加仓后没有同步上移移动止盈，且“怕卖飞”压过“保护利润”，导致高浮盈仓位在板块兑现时从趋势仓退化为保本仓。
- 有时过于聚焦熟悉标的，导致视野不够打开、候选池不够广。
- 对估值高低的判断，如果没有统一口径，容易造成结论失真。
- 对消息、产品、模型或事件的真实性验证较敏感，但还需要进一步前移到“交易前”而不是“持仓后”。

## 复盘中已经明确的改进方向
- 每笔交易都应写清：
  - 为什么买
  - 逻辑通过哪条链路传导
  - 什么情况算证伪
  - 何时减仓或退出
- 不能只复盘成功案例，失败案例和利润回吐案例更有价值。
- 需要把主题研究、标的选择、仓位控制和退出规则拆开单独评估。
- 需要引入更客观、可批量计算的行业景气与拥挤度指标，减少拍脑袋选方向。
- 已开始把“板块拥挤度”拆成固定观察维度：涨速斜率、核心股换手同步抬升、讨论热度扩散、回调时承接强弱。
- 对条件单的要求明显提高：后续应尽量写明触发价位、确认条件、仓位动作和取消条件，避免盘中在技术纪律和主观理解之间临时切换。
- 强趋势票浮盈后必须把退出拆成三层：第一层冲高或事件兑现减仓锁利润，第二层跌破短线强弱位继续降风险，第三层跌破成本/结构失效位无条件退出；加仓后止损不能继续沿用最初买入时的远端支撑。
- 2026-04-15 更正后用交易记录 90 条有效买卖、20 个持仓回合重跑最大公约数回测：当前可执行 V1 规则为 `初始硬止损 -8%`、`收盘浮盈 +7% 卖出 1/2`、`第一次止盈后按最高收盘价回撤 -4% 清/再降风险`；加仓点测试显示 `收盘浮盈 +5%` 是当前样本里较优位置，舒适版加仓量不超过首仓 50%。光迅科技 2026-04 交易更正后当前合计盈亏约 +409 元，不是此前误算的 +2959 元。
- 复盘表格需要严格区分 `股数 / 持仓市值 / 仓位占比`，避免把“仓位”写成市值，导致风险暴露判断失真。

## 协作约定
- 后续协作应优先输出：
  - 结论
  - 证据分级
  - 证伪条件
  - 与持仓或候选标的的直接映射

  - AI / 政策 / 宏观事实汇总
  - 海外产业链变化
  - A 股映射优先级
- 对任何关键论断，优先区分：
  - 已验证事实
  - 高概率推断
  - 纯猜想
- 当用户提出一个行业或个股想法时，应优先帮助其完成：
  - 主题定义
  - 行业景气判断
  - 个股受益层级判断
  - 交易条件与退出条件设定
- 输出格式偏好：交易分析、财报/业绩预告、预期差、空头对抗、执行条件等内容，默认使用 `text` 代码块中的 ASCII 全框线表格（Claude Code 风格），少用普通 Markdown 表格，避免中文、数字、链接混排导致显示不齐。
