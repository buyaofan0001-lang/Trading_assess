# 交易看板数据缺失诊断（2026-07-20 10:23）

## 用户问题

为什么没数据？

## 核对结果

- 本地 LaunchAgent `com.liuguan.trading-assess.dashboard` 正常运行，进程监听 `127.0.0.1:8765`。
- `/api/portfolio`、`/api/intraday`、`/api/risk-model` 均返回 HTTP 200；不是服务宕机。
- `/api/dashboard?timeframe=1d` 返回 `partial=true`，国内行情、日线、ATR 和板块资金流共出现 23 项缺口，统一错误为 `token已过期`。
- `/api/intraday` 仍能通过东方财富/AkShare 回退源取得当日分钟价格，但前端折线需要主行情中的昨收价做“相对昨收”计算；昨收为空时前端按真实性规则不绘图，所以显示“今天尚无可用分钟行情”。
- 风控模型自动重跑也因同一 token 失效而失败，面板保留 2026-07-17 的旧结果并明确报错。
- 用户当前打开的 URL 带 `v=1.10.0`，但服务器日志证明实际加载的是 V1.11 的 `app.js`/`styles.css`；旧查询参数不是本次缺失的主因。

## 结论

根因是项目保存的 Tushare/自建接口凭证已过期，不是看板进程、自动刷新或浏览器缓存故障。恢复数据需要更新有效 token；若要提高可用性，后续还应把昨收、日线和 ATR 增加独立行情回退源，避免单一 token 失效导致整个同行擂台退化为空值。
