# Long-term Memory

## 用户与协作风格
- 晖偏好简洁直接的沟通，但对交易研究要求高标准、强证据、可证伪。
- 我在交易协作中的角色是“交易伙伴”，不是下硬性买卖指令的人；应给方向倾向、置信度、触发条件与证伪条件。
- 只接受做多，不接受做空、融券、反向交易计划。
- 对不确定信息必须明确标注“未证实/待验证”。
- 发现引用或事实错误后，下一轮必须给出更正声明、正确来源、以及受影响结论范围。
- 交易输出尽量表格化，但若用户临时要求改格式，优先服从当轮要求。

## 交易方法与固定流程
- 交易分析默认按固定顺序进行：
  1) 用户提交交易计划
  2) 宏观扫雷
  3) 行业深研
  4) 基本面分析检查
  5) 观点拆解与逐条取证
  6) 空头对抗
  7) 技术面精准定位
  8) 逻辑+技术双维验证
  9) 执行与失效剧本
  10) 事后复盘
- 适用主观交易计划的验证与风险评估；不适用于高频日内、自动化下单、无证据判断。
- 默认更偏向“产业逻辑先行，再找个股映射”，技术面主要服务执行，而不是单独充当核心买入理由。
- 复杂任务默认采用四视角拆解：总控守门、宏观行业、空头对抗、技术执行。

## 证据与信息验证规则
- 交易相关输出必须证据贯穿，不能给无证据结论。
- 关键结论优先要求“双源交叉”：官方/监管/财报公告 + 独立二次来源；冲突时优先官方口径。
- 每条关键信息都应做：
  - 主体一致性校验（代码、公司名、公告主体）
  - 时间与口径校验（TTM/单季/累计、日期、单位）
  - 可访问性校验（链接可打开，不是404/错误页）
- 对术语歧义必须分层：
  - 事件层：是否发生
  - 技术层：是否明确披露训练/推理/部署事实
  - 因果层：是否映射到订单/收入/利润
- 禁止跨层替代：事件成立 ≠ 技术成立；技术成立 ≠ 业绩映射成立。
- 用户给出链接时，必须优先阅读并在当轮给出“支持 / 部分支持 / 不支持”的核验结果。
- 对易失效站点（如 whitehouse、cninfo detail 页）要格外谨慎：优先最终落地页或静态 PDF，而不是参数页。

## 股票数据准确性硬规则
- 凡是股价、涨跌幅、成交额、换手率、历史K线、最高/最低/开收盘等精确行情数据，必须优先使用 AkShare 这类结构化数据源核实。
- 不能直接引用网页搜索摘要、新闻摘要、搜索卡片充当精确行情。
- 若结构化数据源不可用，必须明确说明受限，不能拿搜索摘要冒充准确数据。

## 用户画像与偏好
- 主要关注 A 股，偏中短期到中期的主题投资与趋势交易。
- 偏好证据优先，反感把未经验证的逻辑直接当结论。
- 强调把信息分层：
  - 事件是否真实发生
  - 产业链传导是否成立
  - 是否映射到订单/收入/利润
  - 是否已经被估值计入
- 更接受有明确证伪条件的交易，不接受只有叙事没有退出机制的交易。
- 关注趋势框架，如“该弱不弱就是强”“30日线”“13日线”“趋势第三阶段”。
- 希望最终形成一套可统计检验、可复盘、可迭代的交易系统，而不是靠感觉和故事。
- 更在意“对当前持仓和决策有没有直接影响”，而不是泛泛行业讨论。
- 对短线与中线仓位分层有明确需求，但执行中容易混淆，需要我主动帮忙拆开。

## 高优先级研究主线
- 电力设备 / 电网设备 / 变压器 / 海外电力需求
- 半导体：存储芯片、先进封装 / 封测、设备、材料
- AI 算力基础设施
- AI 主题跟踪：政策宏观、全球云厂商资本开支、海外AI监管与能源约束、A股映射链条
- 新型储能 / 逆变器 / PCS / 电网侧设备
- 光模块 / 光通信 / 海缆 / CPO / 铜缆
- 热管理 / 液冷 / 机器人零部件
- 铜与有色金属产业链
- 宏观政策与两会对行业主线的影响

## 重点跟踪标的与常见研究口径
- 思源电气：高优先级长期跟踪；核心逻辑是海外电网、电力设备出口、全球电力需求扩张。更看重海外需求弹性而非单纯国内逻辑。对其研究时，要重点核：海外订单/收入增速、现金流修复、估值是否已经过度透支。用户习惯把13日线作为加仓触发观察位，但不应机械拿13日线定义全部中期逻辑失效。
- 长电科技：高频交易与复盘标的；核心逻辑是AI芯片、国产算力、先进封装需求映射到本土OSAT。用户会反复追问“方向对，但利润映射是否足够快、足够强”。当前对它的核心纪律是：47元附近看是否重新转强；47元以下更多按弱修复看，不宜再带预设地硬扛。
- 兆易创新：主要作为存储链条映射标的；关注存储景气、受益层级、高端AI存储映射、业绩释放强度。用户倾向等待放量突破密集抛压区后再更稳健介入。
- 亨通光电：关注AI带来的光通信、海缆、出口与订单；历史教训是不能建立在错误低估值前提上。
- 三花智控：关注机器人、液冷、热管理多重映射；需要产业逻辑与价格确认同时成立。
- 德业股份：主板储能高弹性标的；核心要盯欧洲修复、新兴市场扩张、工商储提升、毛利率和现金流。
- 阳光电源：用户会把它视为“超跌但业绩仍硬”的逆变器/储能代表股。
- 英维克：更偏短线区间思路，逻辑在于液冷是数据中心基础设施链条中较硬的一环，但需要注意订单到利润的映射证据仍不能想当然。

## 已沉淀的方法论与偏好结论
- “行业方向对”不等于“这个时点买这只股票也对”。
- 不能把主题叙事直接等同于业绩兑现。
- 研究集成电路时，优先顺序更偏向：设备 > 先进封装/封测 > 部分设计龙头 > 材料 > 制造。
- 研究储能时，不能沿用“只要装机增长，谁都受益”的旧逻辑；要区分设备、系统集成、PCS/逆变器、电网侧项目能力与运营能力。
- 研究铜时，应拆分资源、冶炼、加工、高端材料，不要把“铜概念”混成一类。
- 对两会、政策主线的使用方式：更适合作为观察池和筛选池，不是直接等权买入池；必须继续核验业绩兑现速度。
- 当市场遇到地缘冲击（如油价/战争）时，用户更倾向寻找“被情绪错杀、但基本面未直接受损”的方向，而不是追逐事件涨幅最大的板块。
- 当没有更好的新标的时，空仓/低仓等待也是有效选择；不能因为“没地方买”就继续被动持有弱票。

## 用户交易心理与常见失误
- 容易先相信主题叙事，再去补证据。
- 容易把“产业方向对”误当成“这个时点买这只股票也对”。
- 盈利阶段有时因为害怕卖飞而延迟止盈，导致利润回吐；这点在思源电气上表现得很明显。
- 一旦失效位到了，用户有时会开始和自己协商，用“基本面没变”“可能反弹”“再等等”来对抗原本的交易纪律；这是高危信号。
- 在交易失效之后反向加仓，是用户需要重点防的错误动作：这通常不是确认机会，而是在放大错误。
- 下次若用户再次出现“祈祷反弹、失效后不走、还想补仓摊低成本”的味道，必须直接提醒：这不是分析，是情绪在接管；标的逻辑未必全错，但这笔交易已经做坏了，先执行纪律，再谈看法。
- 在系统性风险来临时，若持仓没有明显走弱，用户会更愿意把它解释为“该弱不弱就是强”，这有时是对的，但仍需配合纪律位验证。
- 有时过于聚焦熟悉标的，导致候选池不够广；用户自己已经意识到这个问题。
- 若估值口径不统一，容易造成结论失真。
- 对消息真伪比较敏感，但验证动作还需要尽量前移到交易前。
- 对同一持仓，容易混淆“中期逻辑仓”和“短线试错仓”；后续应主动帮他分层。

## 已明确的改进方向
- 每笔交易都应写清：为什么买、逻辑如何传导、什么情况算证伪、何时减仓或退出。
- 失败案例和利润回吐案例比成功案例更值得复盘。
- 需要把主题研究、标的选择、仓位控制和退出规则拆开评估。
- 需要引入更客观、可批量计算的行业景气与拥挤度指标。
- 对话备份和复盘要持续积累，但真正写入长期记忆的只保留长期有效信息。
- 同一股票的历史讨论应尽量视为连续上下文，而不是孤立单轮结论。
- 用户受到“交易员成长路径：感觉下单 -> 价格行为 -> 数学统计 -> 系统迭代”这套框架启发很深，目标是把自己的主观交易逐步升级成正期望系统。

## 交易记录与复盘提炼出的行为特征（初步）
- 交易以 A 股为主，持仓周期多为短期到波段，也会围绕少数主线股做中期持有。
- 交易标的集中在科技成长、AI映射、半导体、电力设备、医药、有色等主线。
- 常见做法是分批买入、分批卖出，而不是一次性满进满出。
- 交易记录表中“买入逻辑 / 计划止盈 / 计划止损 / 清仓逻辑”多为空，说明执行记录化仍有提升空间；后续应主动提醒补全这些字段，方便复盘和策略统计。
- 用户倾向先从大趋势或主线行业入手，再映射到个股；近期尤重视电力、AI硬件、存储、封测、储能、铜。
